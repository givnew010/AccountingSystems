تحليل نظام محاسبي ومخزني
==========================

# الواجهات الامامية (Front-end)
---------------------------------
## هيكلة التصميم في الواجهات
-----------------------------
1. يحتوي النظام على الشريط الجانبي (Sidebar)
  - خصائصه:
    - قابل للطي/الفتح.
    - عند الفتح: يعرض الأيقونات مع أسماء الصفحات.
    - عند الإغلاق: يعرض الأيقونات فقط.
    - يحتوي على بيانات المستخدم الحالي (الاسم – الدور – الصورة).
    - يحتوي على زر تسجيل الخروج.
 - الأيقونات تمثل أقسام النظام:
    - الرئيسية
    - المبيعات
      - العملاء
      - فاتورة المبيعات
      - مردود المبيعات
    - المشتريات
      - الموردين
      - فاتورة المشتريات
      - مردود المشتريات
    - إدارة المنتجات
      - التصنيفات
      - الوحدات
      - المنتجات
    - التقارير
      - تقارير المبيعات
      - تقارير المنتجات
      - تقارير الموردين
      - تقارير العملاء
      - التقارير المالية
    - الإعدادات
      - المستخدمين
      - العملات
      - الفروع
      - المستودعات

2. تنقسم كل شاشة في النظام إلى قسمين رئيسيين:
1.1. قسم القائمة (List Section)
- يعرض جميع السجلات المحفوظة المتعلقة بالكيان (مثل قائمة المنتجات، قائمة الوحدات، قائمة الفواتير، قائمة العملاء، قائمة الموردين).
- يحتوي على عناصر تحكم رئيسية:
  - البحث: للبحث عن السجلات باستخدام الاسم أو الرقم أو أي بيانات أخرى مرتبطة.
  - التصفية: عبر قائمة منسدلة لاختيار الحالة (نشط، موقوف، كل الحالات) أو فلترة خاصة (مثلاً حسب التاريخ، التصنيف، أو الفرع).
- أزرار تنقل: (التالي ← السابق) للتنقل بين السجلات.
- عند النقر على أي سجل في القائمة، يتم فتح تفاصيله في قسم المحتوى.
- تكون في جهة اليمين اذا كان النظام عربي في اليسار اذا كان النظام إنجليزي 

1.2. قسم المحتوى (Content Section)
يحتوي على جميع التفاصيل الخاصة بالسجل المحدد، وينقسم بدوره إلى:

أ) أزرار العمليات (Action Buttons)
- جديد: لإنشاء سجل جديد.
- تعديل: لتعديل السجل الحالي.
- حذف: لحذف السجل الحالي (بشروط مثل عدم وجود عمليات مرتبطة).
- طباعة: في الشاشات التي تحتاج إلى طباعة (مثل الفواتير والتقارير).
- ترحيل: لاعتماد العملية محاسبيًا ومخزنيًا (في الفواتير).
- استيراد: في الشاشات الداعمة (مثل المنتجات والعملاء).
- تكون أزرار العمليات في  اعلى قسم المحتوى.

ب) حقول البيانات (Data Fields)
- تختلف حسب نوع الشاشة.
- أمثلة:
  - شاشة العملاء/الموردين: الاسم، الهاتف، العنوان، الرصيد الافتتاحي، نوع الرصيد، الحالة.
  - شاشة المنتجات: الرقم، الاسم، الباركود، التصنيف، تاريخ الانتهاء، ملاحظات.
  - شاشة الوحدات: اسم الوحدة، الوصف.
  - شاشة الفواتير: رقم الفاتورة، التاريخ، العميل/المورد، طريقة الدفع، جدول تفاصيل الأصناف.

ج) الجداول الفرعية (Sub Tables)
- تعرض بيانات مرتبطة بالسجل الرئيسي.
- أمثلة:
  - شاشة المنتجات: جدول الوحدات (اسم الوحدة، النوع، المكافئ، أسعار البيع/الشراء/التكلفة، الكمية الحالية).
  - شاشة الفواتير: جدول الأصناف (اسم المنتج، الوحدة، الكمية، السعر، الخصم، الإجمالي).
  - شاشة التقارير: جدول النتائج بناءً على معايير البحث والتصفية.

3. ملاحظات:
- جميع شاشات النظام لها هيكل موحّد: قسم القائمة + قسم المحتوى.
- الاختلاف يكون فقط في تفاصيل الحقول والجداول حسب نوع الشاشة.
- Sidebar عنصر ثابت على مستوى النظام لإدارة التنقل والصلاحيات.

## االواجهات 
---------------
1. المبيعات
-----------
1.1 فاتورة المبيعات
- رقم الفاتورة: حقل عرض (تلقائي)
- التاريخ: حقل إدخال
- العميل: قائمة منسدلة من جدول العملاء
- طريقة الدفع: قائمة منسدلة (نقدا، آجل، تحويل)
- جدول المنتجات: (اسم المنتج، الوحدة، الكمية، السعر، الخصم، الإجمالي)
- ملاحظات: حقل نصي متعدد الأسطر
- أزرار: إضافة صف، حذف صف، حفظ، طباعة

1.2 فاتورة مردود المبيعات
- نفس فاتورة المبيعات مع إضافة حقل الفاتورة المرجعية

1.3 العملاء
- الحقول: اسم العميل، الهاتف، العنوان، الرصيد الافتتاحي، نوع الرصيد، الحالة
- جدول عرض للعملاء مع التعديل والحذف

2. المشتريات
-------------
- فاتورة المشتريات: مثل فاتورة المبيعات لكن المورد بدلا من العميل وسعر الشراء بدلا من البيع
- فاتورة مردود المشتريات: مثل مردود المبيعات
- الموردين: مثل العملاء

3. إدارة المنتجات
-----------------
3.1 المنتجات
- اسم المنتج، الباركود، التصنيف، المخزن/الفرع، الحالة
- جدول للوحدات:
  - اسم الوحدة، نوع الوحدة (افتراضية/فرعية)، معدل التحويل
  - سعر البيع، سعر الشراء، سعر التكلفة

3.2 التصنيفات
- اسم التصنيف، وصف، جدول عرض

3.3 الوحدات
- اسم الوحدة، وصف، جدول عرض

4. التقارير
------------
- تقرير المبيعات: فلترة بالتاريخ/العميل + جدول عرض للفواتير
- تقرير المنتجات: فلترة بالتصنيف/المنتج/المخزن + جدول عرض للكميات والأسعار

5. الإعدادات
-------------
- المستخدمون: الاسم، البريد، كلمة المرور، الدور، جدول عرض
- العملات: الاسم، الرمز، سعر التحويل، جدول عرض
- الفروع: الاسم، العنوان، جدول عرض
- المستودعات: الاسم، الفرع المرتبط، جدول عرض

# العمليات (Operations)
-------------------------------------
1. الإضافة (Add)
- إنشاء سجل جديد (فاتورة، عميل، منتج...)
- تحديث المخزون (زيادة/نقصان)
- تحديث الحسابات المحاسبية

2. التعديل (Edit)
- إلغاء الأثر السابق (مخزون + محاسبة)
- تطبيق الأثر الجديد

3. الحذف (Delete)
- مسموح فقط للسجلات غير المرتبطة أو غير المرحلة

4. الترحيل (Post)
- إنشاء قيد محاسبي تلقائي
- تسجيل حركة مخزون
- قفل الفاتورة

5. الطباعة (Print)
- إنشاء ملف PDF/HTML يعكس البيانات (بدون تغيير قواعد البيانات)

6. الاستيراد (Import)
- استيراد منتجات، عملاء، موردين، أرصدة افتتاحية من ملفات CSV/Excel

7. الجرد المخزني (Inventory Count)
- إدخال الكمية الفعلية ومقارنتها مع النظام
- تسجيل فرق الجرد كمكسب/خسارة مخزون

8. الإغلاق المالي (Financial Closing) - مستقبلي
- إقفال الفترة المالية
- ترحيل الأرباح والخسائر
