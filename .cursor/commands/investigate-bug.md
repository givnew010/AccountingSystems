# Investigate Bug Command

Follow this sequence for reliable bug investigation.

## Workflow

1. Capture exact symptom and expected behavior.
2. Reproduce with minimal deterministic steps.
3. Identify affected layer (`ui`, `composable`, `service`, `api`).
4. Add temporary diagnostics/logging at likely failure points.
5. Propose root cause hypothesis and validate with evidence.
6. Apply fix and add regression verification.

## Required Output

- Reproduction steps
- Root cause
- Fix summary
- Regression prevention notes
