# New Task Command

Use this command to define and start a new project task with clear scope and execution steps.

## Input Template

- Task title:
- Business goal:
- Requested by:
- Priority: (High/Medium/Low)
- Due date:
- Related module/page:
- Dependencies:

## Execution Workflow

1. Clarify acceptance criteria and expected output.
2. Split the task into small actionable subtasks.
3. Identify technical risks and assumptions.
4. Confirm required files/components to change.
5. Start implementation in incremental steps.
6. Track progress after each completed subtask.

## Required Deliverables

- Implementation plan (short and actionable)
- List of touched files
- Test/verification checklist
- Final completion note with next steps (if any)

## Definition of Ready

- Scope is clear
- Success criteria are measurable
- Dependencies are known or explicitly flagged
