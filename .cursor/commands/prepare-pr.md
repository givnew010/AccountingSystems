# Prepare PR Command

Use this checklist before opening a pull request.

## Checklist

- Confirm scope is focused and matches task objective.
- Run lint/tests relevant to changed files.
- Verify UI changes against design system and core workflows.
- Ensure API/service changes include error handling updates.
- Update docs if behavior/contracts changed.

## PR Description Format

1. Why this change is needed.
2. What behavior changed.
3. Test plan (automated + manual).
4. Risk and rollback notes.
