# Project Progress Report Command

Use this command to generate a clear progress report based on project requirements and task execution status.

## Report Inputs

- Reporting period:
- Project objective:
- Source tasks list:
- Requirements baseline:
- Team/member scope (optional):

## Report Structure

1. Overall status summary
2. Requirements coverage
3. Completed tasks
4. In-progress tasks
5. Pending tasks
6. Blockers and risks
7. Next action plan

## Required Sections

### 1) Overall Status

- Progress percentage (estimated)
- Health status: (On Track / At Risk / Delayed)
- Key milestone achieved in this period

### 2) Requirements Coverage

- Total requirements count
- Implemented requirements
- Partially implemented requirements
- Not started requirements

### 3) Task Breakdown

- Completed: include completion evidence
- In progress: include current step and ETA
- Pending: include priority and dependencies

### 4) Risk and Blockers

- Open blocker description
- Impact level
- Owner
- Proposed resolution and target date

### 5) Next Plan

- Top 3 next tasks
- Immediate technical decisions needed
- Support needed (if any)

## Output Style

- Concise bullets
- Actionable language
- No vague status labels without evidence
