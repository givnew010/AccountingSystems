# Update Task Command

Use this command when an existing task needs edits in scope, priority, solution approach, or delivery timeline.

## Input Template

- Task reference:
- Current status:
- Requested change:
- Reason for change:
- Impacted files/modules:
- New deadline (if changed):

## Update Workflow

1. Compare old scope vs new scope.
2. Mark added, removed, and changed requirements.
3. Re-estimate effort and update execution plan.
4. Identify regressions or dependencies introduced by the change.
5. Update verification plan to match the new scope.
6. Communicate what is done, what changed, and what remains.

## Output Format

- Summary of change request
- Scope diff (added/removed/updated)
- Updated task plan
- Risks and mitigation
- Updated completion estimate

## Quality Guardrails

- Avoid hidden scope creep.
- Preserve completed valid work where possible.
- Keep all changes traceable to a requirement update.
